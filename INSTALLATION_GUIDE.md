# Руководство по установке и тестированию

## Проблема решена ✅

Добавлен **background service worker**, который обрабатывает клик на иконку расширения и открывает боковую панель.

## Что было добавлено

1. **Файл**: `src/background/background.ts`
   - Обработчик клика на иконку расширения
   - Открывает боковую панель через `chrome.sidePanel.open()`

2. **Обновлен**: `manifest.json`
   - Добавлена секция `background` с `service_worker: "background.js"`

3. **Обновлен**: `vite.config.ts`
   - Добавлена сборка background service worker

## Установка расширения

### Шаг 1: Сборка
```bash
npm run build
```

### Шаг 2: Загрузка в Chrome

1. Откройте Chrome и перейдите на `chrome://extensions/`
2. Включите **"Режим разработчика"** (Developer mode) в правом верхнем углу
3. Нажмите **"Загрузить распакованное расширение"** (Load unpacked)
4. Выберите папку `dist` из проекта
5. Расширение должно появиться в списке

### Шаг 3: Проверка установки

Убедитесь, что:
- ✅ Расширение появилось в списке
- ✅ Нет ошибок в консоли
- ✅ Иконка расширения видна в панели инструментов (может быть в меню расширений)

## Тестирование боковой панели

### Тест 1: Открытие панели
1. Нажмите на иконку расширения в панели инструментов
2. **Ожидаемый результат**: Боковая панель открывается справа
3. **Если не открывается**: 
   - Проверьте консоль расширения (chrome://extensions/ → Details → Inspect views: service worker)
   - Убедитесь, что файл `background.js` присутствует в `dist/`

### Тест 2: Выбор файла
1. В боковой панели нажмите **"Выберите PDF файл"**
2. Выберите любой PDF файл
3. **Ожидаемый результат**: 
   - Появляется прогресс-бар
   - Показывается статус "Проверка файла..."
   - Затем "Загрузка файла..." и "Сжатие PDF..."

### Тест 3: Просмотр результатов
1. Дождитесь завершения сжатия
2. **Ожидаемый результат**:
   - Отображается исходный размер файла
   - Отображается размер сжатого файла
   - Отображается процент уменьшения
   - Кнопка "Скачать сжатый PDF" активна

### Тест 4: Скачивание
1. Нажмите **"Скачать сжатый PDF"**
2. **Ожидаемый результат**:
   - Файл скачивается с суффиксом "_compressed"
   - Например: `document.pdf` → `document_compressed.pdf`

### Тест 5: Персистентность панели
1. Откройте боковую панель
2. Перейдите на другую вкладку
3. **Ожидаемый результат**: Боковая панель остается открытой

### Тест 6: Обработка ошибок
1. Попробуйте загрузить не-PDF файл (например, .txt или .jpg)
2. **Ожидаемый результат**: 
   - Появляется сообщение об ошибке
   - Процесс сжатия не начинается

```bash
ls -lh dist/
# Должны быть:
# - background.js
# - sidepanel.js
# - sidepanel.html
# - sidepanel.css
# - manifest.json
# - pdf.worker.mjs
```

### Проверка manifest.json
```bash
cat dist/manifest.json
# Должно содержать:
# - "side_panel": { "default_path": "sidepanel.html" }
# - "background": { "service_worker": "background.js" }
# - "permissions": ["sidePanel"]
```

## Известные проблемы и решения

### Проблема: Иконка не открывает панель
**Решение**: 
- Перезагрузите расширение (chrome://extensions/ → кнопка обновления)
- Проверьте консоль background service worker на ошибки

### Проблема: Панель открывается, но пустая
**Решение**:
- Проверьте, что `sidepanel.html` и `sidepanel.js` в `dist/`
- Откройте DevTools для панели и проверьте ошибки

### Проблема: Ошибка при сжатии
**Решение**:
- Проверьте, что `pdf.worker.mjs` в `dist/`
- Проверьте консоль на ошибки загрузки worker

## Требования

- **Chrome версия**: 114+ (для поддержки side_panel API)
- **Node.js**: 18+ (для сборки)
- **npm**: 9+ (для установки зависимостей)

## Дополнительная информация

- Все операции выполняются локально в браузере
- Файлы не отправляются на внешние серверы
- Временные данные удаляются при закрытии расширения
- Типичное сжатие: 50-80% для PDF с изображениями

## Поддержка

Если возникли проблемы:
1. Проверьте консоль браузера (F12)
2. Проверьте консоль background service worker
3. Убедитесь, что все файлы собраны корректно
4. Попробуйте пересобрать проект: `npm run build`
