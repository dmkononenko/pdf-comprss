# Инструкция по тестированию обновленного compression engine

## Что было изменено

Мы полностью переработали алгоритм сжатия PDF:

**Старый подход (не работал):**
- Использовал только pdf-lib
- Не выполнял реального сжатия изображений
- Результат: ~0% сжатия

**Новый подход (эффективный):**
- Использует pdf.js для извлечения страниц
- Рендерит каждую страницу на Canvas
- Сжимает через Canvas API в JPEG (качество 70%)
- Создает новый PDF с pdf-lib из сжатых изображений
- Ожидаемый результат: 50-80% сжатия для PDF с изображениями

## Параметры сжатия

В `src/popup/popup.ts` настроены следующие параметры:

```typescript
{
  imageQuality: 70,      // Качество JPEG (0-100)
  imageScale: 1.0,       // Масштаб рендеринга (< 1.0 уменьшает разрешение)
  removeMetadata: true,  // Удаление метаданных
  optimizeStructure: true // Оптимизация структуры PDF
}
```

## Как протестировать

1. **Соберите расширение:**
   ```bash
   npm run build
   ```

2. **Загрузите в Chrome:**
   - Откройте `chrome://extensions/`
   - Включите "Режим разработчика"
   - Нажмите "Загрузить распакованное расширение"
   - Выберите папку `dist`

3. **Протестируйте сжатие:**
   - Нажмите на иконку расширения
   - Выберите PDF файл (лучше всего с изображениями)
   - Дождитесь завершения сжатия
   - Проверьте результаты

4. **Ожидаемые результаты:**
   - Для PDF с изображениями: 50-80% уменьшение размера
   - Для текстовых PDF: 10-30% уменьшение размера
   - Для уже сжатых PDF: минимальное уменьшение

## Настройка параметров для лучшего сжатия

Если нужно более агрессивное сжатие, измените параметры в `src/popup/popup.ts`:

```typescript
{
  imageQuality: 50,      // Ниже качество = меньше размер
  imageScale: 0.8,       // Уменьшение разрешения на 20%
  removeMetadata: true,
  optimizeStructure: true
}
```

## Проверка консоли

Откройте DevTools для popup (правый клик на popup → Inspect) и проверьте консоль на наличие ошибок.

## Известные ограничения

- PDF с текстом будет преобразован в изображения (текст нельзя будет выделить)
- Очень большие PDF могут занять много времени на обработку
- Требуется достаточно памяти для рендеринга страниц
